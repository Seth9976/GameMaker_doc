collision_space

This built-in read-only variable returns the collision space that the instance is in. This will be one of the following enum members:

Constant Description Value
colspace.room Not part of any UI layers 0
colspace.ui_view Part of a UI layer with "Game View" set to "Viewports" 1
colspace.ui_display Part of a UI layer with "Game View" set to "Display" 2

This affects the collision checks done within the instance, as an instance can only collide with instances in the same space. This also affects the default behaviour of the instance deactivation functions as by default they will only activate/deactivate instances in the same space, however that can be changed in each function call.

 

Syntax:

collision_space

 

Returns:

Collision Space

 

Example:

instance_activate_all(collision_space);

The above code will activate all deactivated instances that are placed within the same collision space as the calling instance.
